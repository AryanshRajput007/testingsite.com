What is ANSI and SPARC Model ?

The ANSI/SPARC model is a conceptual model for Database Management Systems (DBMS) that was developed by the American National Standards Institute (ANSI) and the Standards Planning and Requirements Committee (SPARC) in the late 1970s.

It provides a standard framework for understanding the components and interactions of a database system. The ANSI/SPARC model is also known as the "three-schema architecture" because it defines three levels of abstraction or schemas within a DBMS.

These three schemas are as follows :-

  • External Schema (View Level) :-
    •

    The External Schema, also known as the View Level, represents the user's view of the database. It defines how individual users or applications perceive and access the data. Each external schema corresponds to a specific user or group of users and provides a customized view of the data that is relevant to their needs. Users interact with the database through these external schemas, which shield them from the complexities of the underlying data organization. We use data Abstraction to hide irrelevant complexities from the user.

  • Conceptual Schema (Logical Level) :-
    •

    The Conceptual Schema, also known as the Logical Level, represents the entire database in a unified, logical manner. It provides an abstract representation of the data, focusing on the overall data structure, relationships, and constraints. The conceptual schema acts as an intermediary between the external schemas and the physical database storage. It helps to achieve data independence by separating the logical representation of data from its physical storage details.

  • Internal Schema (Physical Level) :-
    •

    The Internal Schema, also known as the Physical Level, describes how data is physically stored and organized in the storage media (e.g., disks). It deals with the low-level details of data storage, such as data storage formats, indexing methods, and access paths. The internal schema is specific to the DBMS implementation and interacts directly with the operating system and storage devices.

The ANSI/SPARC model provides a clear separation of concerns, making it easier to design, develop, and maintain complex database systems. It enables data independence, allowing changes to be made to one schema without affecting the others. This model has been influential in shaping the design and implementation of modern database systems, and it remains a fundamental concept in the field of database management.



Data Independence

Data independence is a fundamental concept in Database Management Systems (DBMS) that refers to the ability to make changes to the database's internal structure without affecting the external applications or programs that use the data. It allows for modifications to be made to the database without requiring changes to be made to the applications that access or interact with the data.

There are two types of data independence :-

  • Logical Data Independence :-
    •

    Logical data independence relates to the ability to modify the logical or conceptual schema of the database without impacting the external schema and applications. This means that changes in the database organization, such as adding or removing tables, altering relationships, or modifying data types, should not affect the way applications retrieve or manipulate data.

    For example, if a new attribute is added to a table in the database, applications accessing data through the external schema should continue to function without any code changes.

  • Physical Data Independence :-
    •

    Physical data independence refers to the ability to modify the physical storage and implementation details of the database without affecting the conceptual or external schemas. Changes in the physical storage, such as moving data to different disks or using a different indexing technique, should not impact the way data is accessed and manipulated by applications.

    For instance, if the database is moved from one server to another, the external applications should not require changes to connect and work with the data.
